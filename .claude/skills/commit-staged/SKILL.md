# commit — ステージ済み変更のコミット

ステージされているファイル（`git add` 済みの変更）だけを対象に、差分の内容を読み取って簡潔で的確な日本語のコミットメッセージを作成し、コミットする。

## 手順

1. **ステージ状態を確認する**

   ```sh
   git status --short      # ステージ済み(列1) と 未ステージ(列2) を把握
   git diff --cached --stat # ステージ済みの変更ファイル一覧
   git diff --cached        # ステージ済みの実差分（メッセージはこれだけを根拠にする）
   ```

   - **ステージ済みの変更（`git diff --cached`）のみを対象とする。** 未ステージ・未追跡の変更はメッセージに含めない。
   - ステージ済みの変更が無い場合はコミットせず、その旨を伝えて止まる（`git add` するか対象を確認する）。

2. **差分を読んでメッセージを組み立てる**

   - 差分から「何を・なぜ変えたか」を読み取る。ファイル名の羅列ではなく、変更の意図を1行で要約する。
   - 複数の無関係な変更が混在している場合は、その旨を指摘し、分割コミットを提案する（勝手に分割せず確認する）。

3. **コミットする**

   ```sh
   git commit -m "<メッセージ本文>" -m "Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>"
   ```

   コミット後に `git log --oneline -1` で結果を提示する。

## メッセージ規約（このリポジトリの傾向に合わせる）

- **言語は日本語**を基本とする。
- **1行サマリは50文字程度を目安**に、要点を簡潔に。末尾に句点（。）は付けない。
- 変更が特定ファイルに閉じている場合は、`adapter.py: ...` / `ci.bash: ...` のように
  **`<ファイル名>: <要約>`** 形式を使ってよい（既存履歴の慣習）。
- 複数ファイル横断・機能単位の変更は、`<対象/観点>: <要約>` のように要約を主にする。
- 必要に応じて本文（body）で「なぜ」を補足する。`-m` を重ねるか、ヒアドキュメントで複数行にする。
- コミットメッセージの末尾には必ず以下の Co-Authored-By 行を入れる:

  ```
  Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>
  ```

## 注意

- `git add` を勝手に実行して対象を増やさない。あくまで**既にステージされた変更**をコミットする。
- `--amend` / `push` / ブランチ操作は行わない（依頼された場合のみ別途対応）。
- メインブランチ（`main`）上で直接コミットしようとしている場合は、その旨を伝えて確認する。
