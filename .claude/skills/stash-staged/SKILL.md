# stash — 作業中変更の一時退避と管理

作業ツリーの変更を `git stash` で安全に退避・復元・管理する。退避時は内容を読み取り、後から識別しやすい簡潔な日本語のメッセージを付ける。

## 手順

### 退避する（保存）

1. **現在の状態を確認する**

   ```sh
   git status --short   # 退避対象になる変更を把握
   git diff --stat      # 未ステージの変更概要
   git diff --cached --stat # ステージ済みの変更概要
   ```

   - 退避対象になる変更が無い場合は stash せず、その旨を伝えて止まる。

2. **メッセージを組み立てて退避する**

   - 差分から「何の作業中か」を読み取り、後から探しやすい1行メッセージを付ける。
   - デフォルトは追跡済みファイルのみを退避する。未追跡ファイルも含める必要があれば `-u`、無視ファイルも含めるなら `-a` を使う（必要性が明確なときのみ）。

   ```sh
   git stash push -m "<退避メッセージ>"
   git stash push -u -m "<退避メッセージ>"   # 未追跡ファイルも含める場合
   ```

   退避後に `git stash list` で結果を提示する。

### 復元する

1. **一覧を確認し、対象を特定する**

   ```sh
   git stash list                 # 退避一覧
   git stash show -p stash@{N}     # 中身の差分を確認
   ```

2. **復元する**

   - 通常は `pop`（復元してスタッシュを削除）。退避を残したい場合は `apply`。

   ```sh
   git stash pop stash@{N}     # 復元してエントリを削除
   git stash apply stash@{N}   # 復元するがエントリは残す
   ```

   - **コンフリクトが出た場合は勝手に解決せず**、状況を伝えて指示を仰ぐ。`pop` 時にコンフリクトが起きるとエントリは削除されない点に注意。

### 一覧・参照する

```sh
git stash list                  # 退避一覧
git stash show -p stash@{N}      # 指定エントリの差分
```

### 削除する

破壊的操作。**実行前に必ず対象と内容を提示して確認を取る。**

```sh
git stash drop stash@{N}   # 指定エントリを削除
git stash clear            # 全エントリを削除（特に慎重に）
```

## 注意

- どの stash 番号（`stash@{N}`）を対象にするか曖昧な場合は、`git stash list` を提示して確認する。
- `drop` / `clear` / `pop`（削除を伴う）など失われる操作は、内容を確認してから実行する。
- コンフリクトは自動解決しない。発生したら状況を報告して止まる。
- ブランチ切り替えやコミットなど、依頼外の操作は行わない。
